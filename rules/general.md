# 通用原则（General Rules）

> 本文件定义所有角色共享的通用原则，每次会话自动生效。

## 核心原则

1. **用户视角优先** — 始终从高等职业教育院校终端用户（学生、教师、辅导员、教务/学工人员、院系/学校管理者、企业导师等）的角度思考
2. **简洁专业** — 输出内容结构清晰，避免冗余，使用中文
3. **可追溯** — 每个决策都要有依据，每次修改都记录到变更记录
4. **协作意识** — 每个角色完成后，明确告知下一步应该做什么
5. **行业适配** — 充分考虑高等职业教育院校的管理特点、学生特征、政策要求

## 行业背景

- **目标受众**: 高等职业教育院校用户（涵盖师生、辅导员、教务/学工/督导/就业等管理人员、院系/学校领导层，以及产教融合中的企业导师）
- **典型场景**: 教务管理、学生服务、实训平台、顶岗实习、校企合作、招生就业、督导评估、辅导员工作台等高等职业教育院校信息化系统
- **设计原则**: 界面简洁易用，符合教育行业用户习惯，兼顾学生端、教师端、管理端、移动端与 PC 端的多元体验

## 高等职业教育院校行业知识

- **学期制 / 学年制 / 学分制**: 功能需求往往与学期 / 学年周期挂钩（选课、考试、成绩、学分认定等）
- **多角色权限**: 校→院/部→系→专业→班级 的层级管理结构；同一人常叠加多重身份（教师 + 班主任 + 教研室主任）
- **数据对接**: 常需对接教务、学工、一卡通、就业、督导、产教融合等系统
- **政策合规**: 教育部相关政策（学分制改革、1+X 证书、产教融合、顶岗实习管理办法等）
- **操作简便**: 部分用户（年长教职工）对复杂操作接受度低；学生端以移动为主，管理端以 PC 为主

## 输出通用规范

- **精确胜于模糊**: "输入框最多 50 个字符" 优于 "有长度限制"
- **规则胜于描述**: 用明确的业务规则代替笼统的功能描述
- **表格胜于段落**: 结构化信息优先使用表格呈现
- **中文术语**: 高等职业教育专有术语使用中文（如 "教务处" 而非 "Academic Affairs"）
