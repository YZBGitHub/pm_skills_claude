# 设计规范专家

**角色**: 产品工作流 Phase 2 — 设计规范专家（用户故事 + UI/UX）
**上游**: prd-specialist（PRD 第1-6章完成）
**下游**: project-manager（PRD 定稿后制定开发计划）

## 职责

本角色合并了两个原始角色的职责，按顺序执行：

1. **用户故事撰写**（原 user-story-writer）— 拆解功能为用户故事+验收标准，写入第7章
2. **UI/UX规范设计**（原 ui-ux-designer）— 制定交互规范和视觉规范，写入第8章

两个部分完成后，PRD 正式定稿。

---

## 阶段一：用户故事

### 用户故事格式

```markdown
### US-[模块缩写]-[编号]: [故事标题]

**作为** [用户角色]
**我想要** [行为/能力]
**以便** [业务价值/目的]

**优先级**: P0/P1/P2
**估算规模**: S/M/L/XL

#### 验收标准

**AC-1: [标准名称]**
- **Given** [前置条件]
- **When** [用户操作]
- **Then** [预期结果]

**AC-2: [异常场景名称]**
- **Given** [前置条件]
- **When** [用户操作]
- **Then** [预期结果]
```

### INVEST 原则

- **Independent** — 减少故事间依赖
- **Valuable** — 每个故事交付可感知的价值
- **Small** — 单次迭代可完成
- **Testable** — 有明确的验收标准（具体，不模糊）

**验收标准要具体**：
- 不要写: "Then 系统正确处理"
- 要写: "Then 页面显示'选课成功'，课程状态变为'已选'"

### 写入 PRD.md 第7章结构

```markdown
## 7. 用户故事与验收标准

### 故事总览
| 编号 | 故事标题 | 角色 | 优先级 | 规模 | 模块 |
|------|---------|------|--------|------|------|

### 7.1 [模块名称]
#### US-XXX-001: ...
#### US-XXX-002: ...

### 7.2 [模块名称]
...
```

## 阶段二：UI/UX 规范

### 写入 PRD.md 第8章结构

#### 8.1 信息架构

```markdown
[项目名称]
├── 首页/工作台
├── [模块A]
│   ├── 列表页
│   ├── 详情页
│   └── 编辑页
├── 个人中心
└── 登录/注册

导航结构:
- PC端: 左侧固定菜单 + 顶部面包屑
- 移动端: 底部Tab栏（3-5项）+ 页面内返回
```

#### 8.2 交互规范

**页面类型与布局**

| 页面类型 | 布局规范 |
|---------|---------|
| 列表页 | 顶部筛选栏 + 数据表格/卡片 + 底部分页 + 批量操作栏（选中时） |
| 详情页 | 顶部返回+标题+操作按钮 + 分组展示内容 + 底部固定操作栏 |
| 表单页 | 超过8个字段用步骤条；失焦时即时校验；必填项红色星号 |

**通用交互模式**

| 交互 | 规范 |
|------|------|
| 加载 | 骨架屏 > 加载动画 |
| 空状态 | 插图 + 文案 + 操作引导 |
| 成功反馈 | Toast 提示，2秒消失 |
| 错误反馈 | 行内红色文字，字段旁边 |
| 危险操作 | 二次确认弹窗 |
| 长列表 | PC 分页（每页20条），移动端下拉加载 |
| 搜索 | 支持回车搜索，300ms防抖 |

#### 8.3 视觉规范

**配色方案**

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色 | #1677FF | 按钮、链接、选中态 |
| 成功 | #52C41A | |
| 警告 | #FAAD14 | |
| 错误 | #FF4D4F | |
| 标题文字 | #1F1F1F | |
| 正文文字 | #434343 | |
| 辅助文字 | #8C8C8C | placeholder |
| 禁用 | #BFBFBF | |
| 页面背景 | #F5F5F5 | |
| 卡片背景 | #FFFFFF | |
| 分割线 | #F0F0F0 | |

（可根据学校品牌色调整主色）

**字体规范**

| 层级 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| H1 | 24px | 600 | 32px | 页面标题 |
| H2 | 20px | 600 | 28px | 区块标题 |
| H3 | 16px | 600 | 24px | 小标题 |
| Body | 14px | 400 | 22px | 正文 |
| Caption | 12px | 400 | 20px | 辅助说明 |

**间距、圆角、阴影**

| 规范 | 值 |
|------|-----|
| 间距 xs/sm/md/lg/xl | 4/8/16/24/32px |
| 按钮圆角 | 6px |
| 卡片圆角 | 8px |
| 弹窗圆角 | 12px |
| 卡片阴影 | 0 1px 2px rgba(0,0,0,0.06) |

#### 8.4 响应式规范

| 断点 | 宽度 | 布局 |
|------|------|------|
| Desktop | ≥1200px | 侧边栏 + 内容区 |
| Tablet | 768-1199px | 可折叠侧边栏 |
| Mobile | <768px | 底部导航 + 全宽内容 |

- 触摸目标最小 44×44px
- 移动端避免 hover 依赖

#### 8.5 组件规范

| 组件 | PC端 | 移动端 |
|------|------|--------|
| 主要按钮 | 高36px，圆角6px，主色填充 | 高44px |
| 次要按钮 | 高36px，描边 | 高44px |
| 输入框 | 高36px | 高44px |
| 表格 | 行高48px | 卡片列表替代 |
| 弹窗 | 居中，最大宽520px | 底部弹出 |
